# CLAUDE.md

このファイルは、このリポジトリのコードを扱う際の Claude Code (claude.ai/code) 向けガイダンスです。

## プロジェクト概要

JavaScript で書かれた JavaScript インタープリターです。モダンな ES6+ 構文をターゲットとし、**ツリーウォーク** 実行戦略を採用しています。ソーステキストをトークンにスキャンし、AST に解析した後、AST ノードを直接評価します（バイトコードコンパイルステップなし）。

## コマンド

```bash
# 依存関係のインストール
npm install

# 全テストの実行
npm test

# 単一テストファイルの実行
npx jest src/parser/parser.test.js

# 名前パターンに一致するテストの実行
npx jest -t "binary expression"

# ウォッチモードでテストを実行
npx jest --watch

# JS ファイルでインタープリターを実行
node src/index.js <file.js>

# REPL の起動
node src/index.js
```

## アーキテクチャ

パイプラインは4つのステージで構成され、それぞれ独自のディレクトリに収められています：

```
src/
  lexer/      → レキサー（ソース文字列 → トークンストリーム）
  parser/     → パーサー（トークンストリーム → AST）
  interpreter/→ 評価器（AST → ランタイム値）
  index.js    → エントリポイント：REPL とファイル実行器の接続
```

### レキサー（`src/lexer/`）
生のソーステキストを `Token` オブジェクトのフラットなリストに変換します。各トークンは `type`（`TokenType` 列挙型から）、`lexeme` 文字列、ソース `line` 番号を持ちます。レキサーは文字列・テンプレートリテラルのエスケープ、数値リテラル、すべての ES6+ 区切り文字を処理します。

### パーサー（`src/parser/`）
トークンリストを消費して AST を生成する再帰下降パーサーです。各文法規則はメソッドに対応します（例：`parseExpression`、`parseStatement`）。AST ノードは `type` フィールドを持つプレーンな JS オブジェクトです（例：`{ type: 'BinaryExpression', operator, left, right }`）。

処理すべき主要な ES6+ 構文：`let`/`const`/`var`、アロー関数、クラス、分割代入、テンプレートリテラル、スプレッド/レスト、`for…of`、`import`/`export`（モジュールがスコープに入っていない場合は静的解析のみ）。

### インタープリター / 評価器（`src/interpreter/`）
AST を再帰的にウォークします。主要な概念：

- **Environment（環境）** — スコープの連結リスト。各 `Environment` インスタンスは `Map<string, value>` と外側スコープへのポインターを持ちます。変数ルックアップはチェーンをたどります。
- **Evaluate（評価）** — 中央の `evaluate(node, env)` ディスパッチ関数（またはクラスメソッド）が `node.type` で分岐し、JS ランタイム値を返します。
- **Callable（呼び出し可能）** — ユーザー定義関数は定義時の `Environment` をキャプチャしてクロージャを形成します；組み込み関数はプレーンな JS 関数です。
- **Classes（クラス）** — インスタンスはプロパティマップを持ちます；メソッドルックアップは `LoxClass` スタイルのクラスディスクリプター（またはネイティブ JS オブジェクト）を通じてプロトタイプチェーンをたどります。

### エラー処理
全体で2つのエラークラスが使用されます：

- `LexError` / `ParseError` — フロントエンドステージで throw され、行番号を持ちます。
- `RuntimeError` — 評価器が throw し、問題の AST ノードを持ちます。

エントリポイントはこれら3つすべてをキャッチし、スタックトレースなしで人間が読めるメッセージを表示します。

## 主要な慣習

- AST ノードはプレーンオブジェクトです — クラスインスタンスではなく — シリアライズを簡単にするためです。
- 評価器は可能な限りネイティブな JS 値を返します（`number`、`string`、`boolean`、`null`）。ボックス型（例：ユーザークラスのインスタンス）は `__type__` マーカーを持つプレーンオブジェクトです。
- テストはカバー対象のソースファイルと同じ場所に配置されます（`lexer.test.js` は `lexer.js` の隣など）。
- スナップショットテストは避け、明示的な `expect(result).toBe(...)` アサーションを優先します（失敗の診断が容易になります）。
