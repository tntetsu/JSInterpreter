## 1. 概要

JSInterpreter は JavaScript で書かれた JavaScript インタープリターである。  
モダンな ES6+ 構文（async/await を含む）を実行し、**式単位のステップ実行 API**（ステップイン・ステップアウト・ステップオーバー・ステップバック）を提供することを主目的とする。

### 1.1 背景と目的

通常のデバッガーはバイトコードや JIT コンパイル済みコードを対象とするため、JavaScript の評価プロセスを式レベルで可視化することが難しい。  
本システムはソースコードを AST として解釈し、**全評価ステップをスナップショット配列として記録**することで、任意の粒度でのステップ実行・逆方向実行（ステップバック）を実現する。

### 1.2 システムの位置づけ

```
ユーザー（開発者・学習者）
    │
    ├─── プログラム的 API（JSDebugger クラス）
    │        ↑ 外部ツール・テストコード・IDE 統合
    │
    └─── 対話型 REPL デバッガー（CLI）
             ↑ ターミナルから直接操作
```

## 3. 対応する JavaScript 構文

### 3.1 変数宣言

- `let`、`const`、`var` による変数宣言
- 複数同時宣言（`let a = 1, b = 2`）
- オブジェクト分割代入（`let { x, y } = obj`）
- 配列分割代入（`let [a, b] = arr`）
- デフォルト値付き分割代入（`let { x = 0 } = obj`）

### 3.2 式

| 分類 | 内容 |
|------|------|
| リテラル | 数値（整数・浮動小数点・16進・8進・2進・数値セパレーター）、文字列（エスケープ対応）、真偽値、null |
| テンプレートリテラル | バッククォート文字列、`${式}` による補間（ネスト可） |
| 算術演算 | `+` `-` `*` `/` `%` `**` |
| 比較演算 | `==` `!=` `===` `!==` `<` `>` `<=` `>=` |
| 論理演算 | `&&` `\|\|` `!` `??`（null 合体） |
| ビット演算 | `&` `\|` `^` `~` `<<` `>>` `>>>` |
| 代入演算 | `=` `+=` `-=` `*=` `/=` `%=` `**=` `&&=` `\|\|=` `??=` |
| インクリメント | `++` `--`（前置・後置） |
| 三項演算子 | `条件 ? 真値 : 偽値` |
| typeof / void / delete | 単項演算子 |
| instanceof / in | 二項演算子 |
| オプショナルチェーン | `?.`（オプショナルアクセス・呼び出し） |
| スプレッド / レスト | `...` |
| await | `await 式`（async 関数内） |

### 3.3 関数

- 関数宣言（`function f(a, b) { ... }`）
- 関数式（`const f = function() { ... }`）
- アロー関数（`x => x * 2`、`(a, b) => a + b`、ブロック本体）
- **非同期関数**（`async function f() { ... }`、`async () => ...`、`async x => ...`）
- レスト引数（`...args`）
- デフォルト引数（`x = 0`）
- クロージャ（定義時のスコープをキャプチャ）
- 再帰呼び出し

### 3.4 制御フロー

- `if` / `else if` / `else`
- `while` ループ
- `do...while` ループ
- `for` ループ（初期化・条件・更新）
- `for...of` ループ（イテラブル対象）
- `for...in` ループ（列挙可能プロパティ）
- `break` / `continue`
- `return`（値あり・なし）
- `throw` / `try` / `catch` / `finally`

### 3.5 クラス

- `class` 宣言・式
- `constructor`
- メソッド定義（通常・async）
- `extends`（継承）・`super`
- `new` による インスタンス生成
- 静的メソッド（`static`）
- ゲッター・セッター（`get` / `set`）

### 3.6 オブジェクト・配列

- オブジェクトリテラル（`{ key: value }`）
- 短縮プロパティ（`{ x }`）
- 計算プロパティ名（`{ [expr]: value }`）
- スプレッド（`{ ...obj }`）
- 配列リテラル（`[1, 2, 3]`）
- スプレッド（`[...arr]`）
- プロパティアクセス（`obj.key`、`obj[expr]`）

### 3.7 async/await（同期シミュレーション）

async/await を同期的にシミュレーションする。ネイティブ I/O を含まないコードに対応する。

| 構文 | 動作 |
|------|------|
| `async function f() { ... }` | 呼び出し結果が `JSPromise` として返る |
| `async () => expr` | 同上 |
| `await expr` | `JSPromise` を同期的に解決し値を返す |
| `Promise.resolve(val)` | 即時 fulfilled な `JSPromise` を返す |
| `Promise.reject(reason)` | 即時 rejected な `JSPromise` を返す |
| `Promise.all/allSettled/race/any([...])` | 同期的に解決した結果を返す |
| `new Promise((resolve, reject) => { ... })` | executor を同期実行して `JSPromise` を返す |

**制約**: `fetch`・`setTimeout` などの本物の非同期 I/O を含む場合は動作しない。

### 3.8 その他

- `import` / `export`（構文解析のみ、ランタイム実行なし）
- `debugger` 文（無視）
- 行コメント（`//`）・ブロックコメント（`/* */`）
- 自動セミコロン挿入（ASI）のサポート

## 4. ステップ実行機能（F-02 〜 F-05）

### 4.1 実行粒度

本システムのステップ実行は**式（ノード）単位**である。すべての AST ノードの評価開始（enter）と評価完了（exit）を個別のステップとして扱う。

#### 例：`let x = 1 + 2 * 3;` のステップ列

| ステップ | フェーズ | ノード型 | 値 |
|---------|---------|---------|-----|
| 0 | enter | VariableDeclaration | — |
| 1 | enter | BinaryExpression(+) | — |
| 2 | enter | Literal | — |
| 3 | exit  | Literal | `1` |
| 4 | enter | BinaryExpression(*) | — |
| 5 | enter | Literal | — |
| 6 | exit  | Literal | `2` |
| 7 | enter | Literal | — |
| 8 | exit  | Literal | `3` |
| 9 | exit  | BinaryExpression(*) | `6` |
| 10 | exit | BinaryExpression(+) | `7` |
| 11 | exit | VariableDeclaration | — |

### 4.2 ステップイン（F-02）

**操作**: 次のイベントへ1つ進む。  
**関数呼び出し**: 呼び出し先の内部に入る。  
**フェーズ**: enter・exit を問わず次へ。  
**境界**: 最終イベントでは停止（done = true）。

### 4.3 ステップオーバー（F-03）

**操作**: 現在ノードの評価をまとめてスキップし、exit に着地する。  
- 現在が `enter(N)` → 対応する `exit(N)` へジャンプ
- 現在が `exit` → cursor を1つ進める

### 4.4 ステップアウト（F-04）

**操作**: 現在の関数呼び出しを抜け、呼び出し元へ戻る。  
- callDepth が現在より小さくなる最初の `exit` イベントへジャンプ
- callDepth が 0（トップレベル）の場合はプログラム末尾へ

### 4.5 ステップバック（F-05）

**操作**: 1つ前のイベントへ戻る。常に O(1)。  
**精度**: 環境スナップショットはディープクローンで記録されるため、オブジェクト・配列の内部変更も正確に復元できる。

## 5a. ヒューマンフレンドリーステップ（F-12）

### 5a.1 目的

式単位のステップ実行（F-02〜F-05）はすべての AST ノード評価を個別のイベントとして扱うため、`let x = 1 + 2 * 3` 1行だけで 12 イベントが発生する。これはデバッガーとして正確だが、人間がアルゴリズムをノートに追跡するには情報過多になる。

F-12 は**「意味のある変化点」単位**のステップを提供する。人間が手でトレースするときに書き留める情報だけを表示する粒度である。

### 5a.2 対象イベント

以下のイベントを対象とする（`callDepth` が増加しないネイティブ関数呼び出しはスキップ）:

| ラベル | フェーズ | ノード型 | 説明 |
|--------|---------|---------|------|
| 宣言 | exit | `VariableDeclaration` | `let`/`const`/`var` 宣言の完了 |
| 代入 | exit | `AssignmentExpression` | 代入・複合代入の完了 |
| 更新 | exit | `UpdateExpression` | `i++`、`--j` など |
| return | exit | `ReturnStatement` | `return` の実行 |
| throw | exit | `ThrowStatement` | `throw` の実行 |
| 呼出（前） | enter | `CallExpression` | **ユーザー定義関数**の呼び出し直前（関数式のハイライト） |
| 呼出（入） | enter | 関数本体の最初のステートメント | 関数に入った直後（最初の文の先頭） |
| 呼出（後） | exit | `CallExpression` | **ユーザー定義関数**の呼び出し完了（戻り値確定） |
| 条件 | exit | 条件式 | `if`・`while`・`do...while`・`for`・`? :` の `true`/`false` 決定点 |

**ネイティブ関数**（`Math.floor()`・`arr.push()` など）は `callDepth` を増加させないためスキップされる。

### 5a.3 条件テストの検出方法

| 親ノード | 検出方法 |
|---------|---------|
| `IfStatement` | `trace[enterIdx + 1]`（最初の子 = テスト式）の exit（1回のみ） |
| `ConditionalExpression` | 同上 |
| `WhileStatement` | ループ範囲内の `depth+1` exit で `BlockStatement` 以外（イテレーションごとに捕捉） |
| `DoWhileStatement` | 同上 |
| `ForStatement` | `depth+1` exit で `VariableDeclaration`（init）・`BlockStatement`（body）以外 |

### 5a.4 ステップ数の比較

`bubbleSort([3, 1, 2])` の場合:

| モード | ステップ数 |
|--------|-----------|
| 式単位（stepIn） | 〜400 |
| 文単位 | 〜50 |
| **ヒューマンフレンドリー（F-12）** | **〜20** |

### 5a.5 表示フォーマット

```
[宣言  ] line   3  for (let i = 0; i < n - 1; i++) {
[条件  ] line   3  for (let i = 0; i < n - 1; i++) {   →  true
[条件  ] line   6  if (arr[j] > arr[j + 1]) {           →  true
[宣言  ] line   7  const tmp = arr[j];
[代入  ] line   8  arr[j] = arr[j + 1];                 →  1
[代入  ] line   9  arr[j + 1] = tmp;                    →  3
[更新  ] line   5  for (let j = 0; j < n-1-i; j++) {   →  0
```

形式: `[ラベル] line NNN  <ソース行（45文字まで）>  →  値`

値の表示: AssignmentExpression・UpdateExpression・CallExpression・条件式のみ。  
VariableDeclaration・ReturnStatement・ThrowStatement は値を表示しない（ソース行が自明なため）。

### 6.4 CodeTrace — 実行可視化ツール（F-13）

```
ビルド: npm run build:web
起動:   npm run dev:web   → http://localhost:8000
```

初学者向けのブラウザ上で動作する実行可視化ツール。インタープリターのコアは esbuild で `web/interpreter.bundle.js` にバンドルされ、サーバーサイドは不要です。

**レイアウト：**

```
┌──────────────────────────┬────────────────────────┐
│  Source（左 55%）        │  Controls（右 45%）    │
│                          ├────────────────────────┤
│  編集モード: テキスト    │  Current Step          │
│  デバッグモード: 行強調  ├────────────────────────┤
│  + 式ハイライト          │  Variables             │
│  + トレース表（ON時）    ├────────────────────────┤
│                          │  Call Stack            │
│  [step N / 合計]         ├────────────────────────┤
│                          │  Console               │
└──────────────────────────┴────────────────────────┘
```

**コントロールとキーボードショートカット：**

| ボタン | キー | 操作 |
|--------|------|------|
| Step In | `n` / Enter | stepIn() |
| Step Over | `v` | stepOver() |
| Step Out | `o` | stepOut() |
| Step Back | `b` | stepBack() |
| Human Step | `h` | humanStep() |
| Human Back | `H` | humanStepBack() |
| Continue | `c` | continue() |
| Reset | `r` | 編集モードに戻る |
| 📊 Trace | `t` | インライントレース表のON/OFF |

**Source パネル**：現在評価中の行を背景色で強調。さらに式ノードの場合は列範囲を黄色でハイライトし、どの部分式が評価されているかを可視化する。

**Current Step カード**：phase（enter/exit）・nodeType・行:列・depth・callDepth・評価値（exit イベントのみ）を表示。

**Variables カード**：デフォルトは「全スコープの変数をマージして表示（内側スコープが優先）、組み込みグローバル名を除外」。「スコープ別」チェックでスコープチェーンをフレームごとに区分して表示（`Math`・`console` 等の組み込みグローバルも含む）。

**Call Stack カード**：各フレームに関数名・呼び出し位置・**引数の実際の値**を表示（例：`fib(5)`、`bubbleSort([3,1,2])`）。引数の値は呼び出し時点でディープクローンされるため、後の変更の影響を受けない。

**Console カード**：`console.log()` / `console.warn()` / `console.error()` の出力を表示。現在の cursor 位置までに出力された分だけが表示されるため、ステップバック時に出力も遡る。出力は Node.js 互換フォーマットを採用：最上位の文字列引数はクォートなしで表示、配列・オブジェクト内にネストした文字列はシングルクォート付きで表示（例: `console.log(['aaa'])` → `[ 'aaa' ]`）。`JSPromise` は `Promise { value }` または `Promise { <rejected> reason }` 形式で表示される。

**インライントレース表**：`📊 Trace` ボタン（またはキー `t`）で ON/OFF する。ON にすると Source パネルがテーブル形式に切り替わり、各ソース行の右側に変数列と条件式列が追加される。

- **変数列**（青ヘッダー）：実行中に登場したユーザー変数を登場順で列追加する。
- **条件式列**（紫ヘッダー）：`if` / `while` / `for` のテスト式を自動検出し、その評価結果（true/false）を列表示する。条件テキストはソース位置情報（`loc`/`end`）から切り出す。
- **値の更新**：各行が最後に実行された時点のスナップショットを保持し、ステップごとにリアルタイム更新する。
- **変化アニメーション**：cursor が進んだときに変化したセルを黄色（変数）または紫（条件式）でフラッシュ表示する。
- **横スクロール**：変数・条件式が多い場合は Source パネル全体を横スクロールできる。行番号列は固定（`position: sticky`）。

**サンプルプログラム**（ドロップダウン）：フィボナッチ・階乗・バブルソート・クロージャ・クラス。

## 8. 制約・既知の制限

### デバッガー固有

| 制約 | 内容 |
|------|------|
| 無限ループ | `maxSteps` オプションで上限設定可（デフォルト: 100,000 ステップ） |
| ステップバックの精度 | `Map`・`Set` 等のネイティブオブジェクトは参照保持のため変更履歴が不正確になる場合がある |

### 未対応の構文

| 構文 | 状況 |
|------|------|
| 正規表現リテラル | `/pattern/` 構文は非対応（`RegExp` コンストラクターは使用可） |
| `switch` 文 | 未実装 |
| ラベル付き文 | 未実装 |
| `with` 文 | 未実装（非推奨構文） |
| `function*` / `yield` | ジェネレーターは実行不可 |
| タグ付きテンプレートリテラル | 未実装 |
| `for await...of` | 未実装 |

### ランタイムの制限

| 制限 | 内容 |
|------|------|
| ネイティブ async I/O | `fetch`・`setTimeout` 等は動作しない |
| ネイティブメソッドへの JSFunction 渡し | `[1,2,3].map(x => x*2)` 等は動作しない |
| `arguments` オブジェクト | 未対応（レスト引数 `...args` で代替） |
| モジュール | `import/export` は構文解析のみ |
